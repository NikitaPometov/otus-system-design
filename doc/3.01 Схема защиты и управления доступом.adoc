=== Схема защиты и управления доступом

Доступ извне по HTTPS (TLS 1.3) (HTTP/3 по возможности) до API GW / LB. WAF в
 Istio Ingress Gateway в конфигурации reverse-proxy.

Доступ со стороны CDN ограничен на уровне сетевых политик (Allowlist IP).

.Data Leak Prevention
Инспекция ответов на Gateway: блокируется
возврат наборов записей ПДн в одном ответе без явной бизнес-причины
(защита от bulk extraction). Действия привилегированных ролей
(admin, support) над ПДн записываются в аудит-лог.

==== AuthN
OAuth 2.1 + OIDC с JWT access token (TTL: 15 минут) + refresh
token (TTL: 2 месяца).
Shadow Accounts для guest checkouts. В качестве ключа используется способ
подтверждения заказа (Users.user_identifier). После регистрации полноценный
пользователь может привязать произвольное количество бывших теневых аккаунтов
 соответствующими способами подтверждения.

Мотивировать пользователей использовать многофакторную аутентификацию: TOTP с
recovery codes, passkey, OTP по различным каналам.

Для ролей supervisor, support, admin многофакторная аутентификация
обязательна (passkey).

==== AuthZ
RBAC через OPA sidecar на каждом сервисе.
Роли: guest, customer, supervisor, support, admin.
