=== Схема защиты и управления доступом

Доступ извне по HTTPS (TLS 1.3) (HTTP/3 по возможности) до API GW / LB.

AuthN: OAuth 2.1 + OIDC с JWT access token (TTL: 15 минут) + refresh
token (TTL: 2 месяца). Shadow Accounts для покупки без создания профиля.

Мотивировать пользователей использовать многофакторную аутентификацию: TOTP с
recovery codes, passkey, OTP по различным каналам.

AuthZ: RBAC через OPA sidecar на каждом сервисе. В случае сложных или
динамических политик — S3 с OPA Bundle API.

mTLS между сервисами (под управлением Istio)

Секреты для микросервисов — в Vault.
