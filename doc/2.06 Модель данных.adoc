=== Модель данных

Модель описывает логические сущности и связи между ними. Типы полей и
физические детали хранения приведены в `2.14 Схемы данных`.

==== Users

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`user_id`
|Идентификатор пользователя.

|`name`
|Имя пользователя.

|`identifier`
|Идентификатор для поиска и идентификации (`type`, `value`), уникален в рамках системы.

|`restrictions`
|Ограничения пользователя: флаг бана, причина, дата бана.

|`is_deleted`
|Флаг удаления пользователя.

|`audit`
|Служебные поля времени создания/обновления.
|===

==== Goods

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`goods_id`
|Идентификатор товара.

|`name`
|Название товара.

|`description`
|Описание товара.

|`category`
|Категория товара.

|`price`
|Цена (`currency`, `units`).

|`stock`
|Текущий остаток и признак активности карточки.

|`attributes`
|Набор произвольных характеристик товара (`key -> value`).

|`media`
|Связанные изображения/миниатюры и их метаданные.
|===

==== Carts

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`cart_owner`
|Владелец корзины (`user_id` или `external_id`, либо `cart_id` для гостя).

|`items`
|Позиции корзины: `goods_id`, количество, снэпшот названия и цены.

|`updated_at`
|Время последнего обновления корзины.

|`ttl`
|Срок жизни корзины в хранилище.
|===

==== Orders

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`order_id`
|Идентификатор заказа.

|`customer`
|Покупатель (`user_id` или `external_id`).

|`status`
|Статус жизненного цикла заказа.

|`dates`
|Дата создания и дата завершения.

|`address`
|Адрес доставки (нормализованный адрес + координаты + `fias_id`).

|`payment`
|Состояние оплаты, способ оплаты, провайдер/токен.

|`total_amount`
|Итоговая сумма заказа.

|`idempotency_key`
|Ключ идемпотентности создания заказа.

|`cancelled_reason`
|Причина отмены заказа.
|===

==== OrderItems

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`order_id`
|Ссылка на заказ.

|`goods_id`
|Ссылка на товар.

|`quantity`
|Количество единиц товара.

|`price_snapshot`
|Снэпшот цены на момент заказа (`unit_price`, `line_total`).
|===

==== OrderStatusHistory

[cols="1,2",options="header"]
|===
|Атрибут |Состав/назначение

|`order_id`
|Ссылка на заказ.

|`status`
|Статус на момент времени.

|`timestamp`
|Время перехода в статус

|===

==== Связи объектов

[cols="1,2,2",options="header"]
|===
|Связь |Кардинальность |Описание

|`Orders.customer_user_id -> Users.user_id`
|`N:1` (опционально)
|Заказ может принадлежать зарегистрированному пользователю.

|`OrderItems.order_id -> Orders.order_id`
|`N:1`
|Позиция заказа всегда принадлежит одному заказу.

|`OrderStatusHistory.order_id -> Orders.order_id`
|`N:1`
|История описывает один заказ.

|`OrderItems.goods_id -> Goods.goods_id`
|`N:1` (логическая)
|Позиция заказа ссылается на товар каталога.

|`Goods.attributes -> Goods.goods_id`
|`N:1`
|Характеристики принадлежат товару.

|`Goods.media -> Goods.goods_id`
|`N:1`
|Медиа-объекты принадлежат товару.

|`Carts.items.goods_id -> Goods.goods_id`
|`N:1` (логическая)
|Элемент корзины ссылается на товар каталога.
|===

==== Ограничения целостности на уровне модели

[arabic]
. У заказа заполнен ровно один идентификатор покупателя: `user_id` или `external_id`.
. `OrderItems` не существуют без соответствующего `Order`.
. Количество в `OrderItems` и `Carts.items` неотрицательно и не равно нулю.
. Остаток товара неотрицателен.
. Для каждого пользователя комбинация (`identifier.type`, `identifier.value`) уникальна.
